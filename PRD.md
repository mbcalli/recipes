# Recipe App

This is a product requirements document for a recipe management application. This application is intended to help me better manage my pantry inventory and plan meals for my family.

# Goal

A user can collect recipes and send them to the tool. The tool can extract the ingredients and instructions. The tool can plan a week's meals for the user. The tool can adapt to user requests. The tool can manage a user's panty inventory.


# Non-negotiables

The tool must be usable by a lay person. The tool must be adaptive. The tool must not omit important recipe information.

# Tech Stack & Existing Patterns

I am a Python developer. However, use the appropriate tech stack.

# Scope Boundaries
## In Scope

A tool that can recieve recipes, extracts the ingredients and the instructions, populates an internal database, serves recommendations to the user, and can take user feedback.

## Out of Scope

For now, a robust user interface is out of scope.
